Thinking like a computer can be one way of the many ways of describing computational thinking

Computational thinking is the process in which we break down a larger complex problem into smaller subproblems so that it can be better understood, analyzed so that a possible solution can be developed, it can be broken down into 4 stages Decomposition, Pattern recognition, Abstraction, Algorithm.

Decomposition is the process of breaking a big problem into smaller ones so that it is easier to understand and, therefore, easier to solve.
Pattern recognition looks for similarities between the subproblems to find a better solution. For example, if two subproblems have the same pattern, it is more likely that their solutions will be the same
Abstraction is the process of getting rid of everything except the most important parts of a problem, which leaves us with the best and quickest solution.
An algorithm is a detailed set of steps for how to solve a problem.

We use computational thinking in our daily lives more often than we know. For example, in studying, for example. In this module itself, our subject is broken into different chapters (decomposition), then we find a way to relate each chapter to one another (pattern recognition), then only the information relevant to the course as well as level of study is taken out while the rest is discarded (abstraction), and the way we consume this information is then thought out by different lectures in a systematic order so that the student best grasps all the information (algorithm).

Without the use of computational thinking, there would be a significant amount of irrelevant content that would be absorbed, resulting in either an overflow of information for students that may or may not be appropriate for their level of study. Not only that, but even the lecturer would struggle to gather and determine what information to transfer to the students. It ultimately makes learning more difficult. Additionally, some teaching modules that make use of diagrams and flowcharts would be quite challenging.

Computational thinking is used in your careers as well, for example I aspire to become a video-game programer in which computational skills are used everywhere. The three stage of video game development are pre production, production and post productions which forms a simple algorithm about how a video game is produced Pre-production stage which tells us what the game is and how to make it what our target audience is, etc. Over here we are dividing the project into several sub projects as well as identifying certain relations form pervious gaming project, for example if it's is a sequel then a lot of the concepts would be similar, this gives us an initial idea of what the game is gonna be and what all concepts needs to be programmed in order for us to receive the end product. To production in which we build algorithms for each subproject and see the relationship and the interaction between each project , for example how level design interacts with game physics or how the concept arc relates with the level design. And finally the last stage which is post-production in which we extract data about how well the game did, any bug fixes required as well as if more bonus content is required. (Stefyn 2022)
